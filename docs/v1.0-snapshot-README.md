A public, queryable index of NCBI SRA, GEO, BioSample, BioProject, PubMed, and EBI BioSamples metadata, plus a derived publication↔accession linkage table mined from PMC OA full text.

## Quickstart

The Parquet files are the contract. They can be read by any Parquet-aware engine. Examples below.

### DuckDB

The shortest path. The `views.sql` helper defines named views over the published URLs so you can query by table name immediately.

```bash
duckdb -c ".read https://data-omicidx.cancerdatasci.org/v1.0/views.sql; SELECT COUNT(*) FROM sra"
```

Or from a script:

```sql
.read https://data-omicidx.cancerdatasci.org/v1.0/views.sql

-- Find all GEO series mentioning "single-cell RNA-seq" in their title
SELECT gse, title
FROM geo
WHERE title ILIKE '%single-cell RNA-seq%'
LIMIT 10;

-- Cross-source join: BioProject → linked publications
SELECT bp.accession, bp.title, pal.pmid
FROM bioproject bp
JOIN publication_accession_linkage pal
  ON pal.accession = bp.accession
WHERE pal.mention_type = 'deposition'
LIMIT 25;
```

### Polars (Python)

```python
import polars as pl

base = "https://data-omicidx.cancerdatasci.org/v1.0"

sra = pl.read_parquet(f"{base}/sra.parquet")
print(sra.shape)
print(sra.columns)

# Lazy scan for large tables — pushes column projection to the reader
geo = (
    pl.scan_parquet(f"{base}/geo.parquet")
    .filter(pl.col("title").str.contains("single-cell"))
    .select(["gse", "title"])
    .head(10)
    .collect()
)
```

### DataFusion (Python)

```python
from datafusion import SessionContext

ctx = SessionContext()
ctx.register_parquet("sra", "https://data-omicidx.cancerdatasci.org/v1.0/sra.parquet")

ctx.sql("SELECT COUNT(*) FROM sra").show()
```

### R (`arrow`)

```r
library(arrow)

base <- "https://data-omicidx.cancerdatasci.org/v1.0"

# Streaming read — does not load the whole file
sra <- open_dataset(paste0(base, "/sra.parquet"), format = "parquet")
sra |>
  dplyr::filter(grepl("single-cell", title, ignore.case = TRUE)) |>
  dplyr::select(study_accession, title) |>
  head(10) |>
  dplyr::collect()
```

### pandas + pyarrow

```python
import pyarrow.parquet as pq
import pandas as pd

url = "https://data-omicidx.cancerdatasci.org/v1.0/biosample.parquet"

# Read only the columns you need — pyarrow pushes projection down
df = pq.read_table(url, columns=["accession", "organism", "submission_date"]).to_pandas()
print(df.head())
```

## Schema

The detailed schema for each table is documented at `{{DOCS_URL}}`. The schema is **frozen as v1.0** at this snapshot (see ADR-0002 and ADR-0003 in the repository). Future schema changes are versioned, not retroactive.

Each row in `publication_accession_linkage` includes the accession's canonical form, the linked PMID/PMCID, the section the mention was found in (Title / Abstract / Methods / Data Availability / Results / etc.), a short context window (prefix / exact / postfix), a derived mention-type class (deposition / reuse / comparison / passing), and the per-article source license tag. See ADR-0001 for the architectural decision behind this layout.

## Versioning

OmicIDX uses semver-adjacent versioning (see ADR-0003):

- **Major (`vN.0`)** — Breaking changes (column removal, type change, canonical-form rule change)
- **Minor (`v1.x`)** — Additive changes (new columns, new tables, new sources)
- **Patch (`v1.0.x`)** — Bug fixes that do not change schema shape or semantics

Future snapshots will appear at sibling prefixes: `s3://data-omicidx/v1.1/`, `v1.2/`, etc.

**Preservation, not time-travel.** Past snapshots are not deleted from the bucket. However, queryability of older snapshots may degrade as query engines and upstream sources evolve. Consumers needing long-term legacy access should archive snapshots they care about at their own end.

## Reproducibility

Every snapshot is generated from a versioned commit of the repository. The commit hash that produced this snapshot is recorded at `{{REPO_URL}}/releases/tag/v1.0`.
